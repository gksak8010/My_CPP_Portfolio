1. 프로젝트 개요
  c++의 객체 지향 프로그래밍 기초와 포인터, 동적 메모리 관리를 학습하고 적용하기 위해 구현한 콘솔 기반의 학생 성적 관리 프로그램입니다.
  사용자는 이 프로그램을 통해 학생의 이름, 학년, 반, 성적 정보(국어, 영어, 수학)를 추가, 조회, 수정, 삭제 하는 기본적인 기능을 수행할 수 있습니다.

3. 주요 기능
   학생 정보 신규 등록(이름, 학년, 반, 국어, 수학, 영어)
   총 저장 학생 리스트 출력
   이름으로 학생 찾기
   이름으로 학생을 찾고 정보 수정
   이름으로 학생을 찾고 정보 삭제

4. 적용된 핵심 기술
   
   객체지향 설계
   -Student 클래스에 학생의 개인 정보(이름, 성적 등)와 관련 동작(정보 출력, 수정)을 캡슐화했습니다.
   -StudentContainer 클래스를 별도로 설계하여, Student 객체들을 관리하는 책임을 분리했습니다.

   동적 메모리 할당 및 관리
   -학생 정보가 추가될 때마다 new 키워드를 사용하여 Student 객체를 힙(Heap) 메모리에 동적으로 생성했습니다.
   -학생 정보가 삭제될 때 delete 키워드로 해당 객체의 메모리를 명확하게 해제하여, 메모리 누수를 방지하고자 했습니다.

   포인터 배열을 이용한 데이터 관리
   -student 객체 자체를 배열에 저장하는 대신, Student 객체를 가리키는 포인터를 배열에 저장했습니다.
   -이를 통해 객체 생성 및 관리에 유연성을 더하고 C++의 포인터 개념을 심도 있게 활용했습니다.

   데이터 검색 및 삭제 로직 구현
   -선형 탐색 알고리즘을 구현하여 사용자가 입력한 이름과 일치하는 학생을 찾았습니다.
   -특정 학생 데이터 삭제 시, 단순히 delete로 메모리를 해제하는 것에서 그치지 않았습니다. 배열의 연속성을 유지하고 추후 접근 오류를 막기 위해, 삭제된 요소 뒤의 모든 포인터를 한 칸씩 앞으로 재배치하는 로직을 구현했습니다.

5. 배운 점 및 응용
   C++에서 new와 delete를 사용한 동적 메모리 관리의 중요성과 메모리 누수 및 접근 오류의 위험성을 명확히 이해했습니다.
   C 스타일의 문자열 대신 C++의 std::string을 사용하면 문자열 비교 및 관리가 훨씬 안전하고 간편해진다는 것을 배웠습니다.
   현재는 고정 크기 배열을 사용했지만, std::vector와 같은 C++ 표준 라이브러리의 컨테이너를 활용하면 크기 제한이 없고 데이터 삽입/삭제가 훨씬 효율적인 프로그램을 만들 수 있음을 알게 되었습니다.
